# A 역할 2주차 작업 정리

2주차 목표는 3D 아바타가 신체 정보와 포즈 선택에 반응하도록 만드는 것입니다.

## 현재 구현 상태

- `poses.js`에 6개 포즈 프리셋을 정의했습니다.
  - 기본 서기
  - T-포즈
  - 측면
  - 걷기
  - 앉기
  - 자신있게
- 리깅된 GLB가 들어오면 `boneRotations` 값을 실제 본에 적용합니다.
- 현재처럼 뼈가 없는 `NonRig` GLB가 들어오면 `fallback` 회전/위치값으로 캡처용 포즈 차이를 만듭니다.
- `boneMap.js`는 Ready Player Me 계열 이름과 Mixamo 계열 `mixamorig*` 이름을 함께 찾도록 확장했습니다.
- `Avatar.jsx`는 GLB 크기와 원점이 달라도 화면 중앙에 맞도록 자동 정렬합니다.

## 중요한 제한

현재 `public/avatars/male.glb`는 `skins: 0`, `bones: 0`, `animations: 0`인 non-rig 모델입니다.

따라서 지금 파일로는 화면 표시, 회전, 캡처용 각도 변경은 가능하지만 팔/다리 관절을 정확히 움직이는 포즈는 불가능합니다.

2주차 발표/시연 품질을 높이려면 Ready Player Me 또는 Mixamo 호환 리깅 GLB를 다시 받아서 같은 경로에 교체해야 합니다.

```text
public/avatars/male.glb
```

## 리깅 GLB 교체 후 확인할 것

1. 브라우저 개발자 도구 콘솔에서 `=== Avatar bones ===` 아래 본 이름이 출력되는지 확인합니다.
2. `Hips`, `Spine`, `LeftArm`, `RightArm`, `LeftUpLeg`, `RightUpLeg` 또는 `mixamorig*` 계열 이름이 보이는지 확인합니다.
3. 포즈 버튼을 눌렀을 때 팔/다리 각도가 바뀌는지 확인합니다.
4. 신체 정보 슬라이더를 움직였을 때 허리, 가슴, 엉덩이 비율이 변하는지 확인합니다.
5. `옷 입혀보기`를 눌러 현재 포즈 캡처가 갤러리에 들어가는지 확인합니다.
